# API

This document describes the Next.js route handlers implemented in this repository and the client-side usage patterns.

## Base URL
- Local dev: `http://localhost:3000`
- No external backend is required for the current MVP.

## Endpoints

### GET `/api/live-sync`
Returns a snapshot of Kaseya and Strev assets, plus summary stats. Uses live APIs if configured; otherwise returns mock data.

**Response (JSON)**
```
{
  "ok": true | false,
  "checkedAt": "ISO-8601 timestamp",
  "source": "live" | "mock",
  "summary": {
    "kaseyaCount": number,
    "strevCount": number,
    "outOfSyncCount": number,
    "matchedCount": number
  },
  "kaseyaAssets": [
    {
      "id": "string",
      "name": "string",
      "identifier": "string",
      "modifiedDate": "ISO-8601 timestamp",
      "hasAssetInfo": boolean,
      "assetInfoCount": number
    }
  ],
  "strevAssets": [
    {
      "id": "string",
      "name": "string",
      "identifier": "string",
      "modifiedDate": "ISO-8601 timestamp"
    }
  ],
  "message": "string (optional)"
}
```

**Behavior**
- If live fetch fails, `source` is `mock`, `ok` is `false`, and `message` describes the failure.
- The dashboard sync panel and assets page rely on this endpoint.

**Implementation**
- `src/app/api/live-sync/route.ts`
- Live data source: `src/lib/live-sync-data.ts`

---

### GET `/api/connectivity/test`
Runs connectivity checks against Kaseya and Strev APIs.

**Query Parameters**
- `target=strev` to test Strev only
- `target=kaseya` to test Kaseya only
- Omit `target` to test both

**Response (JSON)**
```
{
  "results": [
    {
      "service": "strev" | "kaseya",
      "ok": true | false,
      "statusCode": number (optional),
      "message": "string",
      "checkedAt": "ISO-8601 timestamp"
    }
  ]
}
```

**Behavior**
- Strev test uses `REVNUE_TEST_URL` + `REVNUE_TOKEN`.
- Kaseya test uses `KASEYA_TOKEN_ID` + `KASEYA_TOKEN_SECRET` and the resolved assets URL.
- When Kaseya fetch fails, it attempts a host probe for a clearer message.

**Implementation**
- `src/app/api/connectivity/test/route.ts`

## Client Usage

### Assets Page
- Calls `GET /api/live-sync` on mount
- Builds compare rows and resets in-memory transfer history
- File: `src/app/(console)/assets/page.tsx`

### Dashboard Sync Panel
- Calls `GET /api/live-sync` on mount and on refresh button
- File: `src/components/dashboard/dashboard-sync-panel.tsx`

### Support Page
- Connectivity tests are triggered from the UI and call `GET /api/connectivity/test`
- File: `src/components/transfers/connectivity-test-card.tsx`

## Related Environment Variables
Used by the API routes and live data fetchers:
- `REVNUE_ASSET_URL`
- `REVNUE_TEST_URL`
- `REVNUE_TOKEN`
- `REVNUE_COMPANY`
- `KASEYA_BASE_URL`
- `KASEYA_ASSET_URL`
- `KASEYA_ASSETS_URL`
- `DEFAULT_KASEYA_ASSETS_URL`
- `KASEYA_TOKEN_ID`
- `KASEYA_TOKEN_SECRET`
- `DEFAULT_USER_AGENT`

## Notes
- No write APIs are implemented in this repo (transfer execution, settings persistence, etc.).
- All API responses are generated by Next.js route handlers at runtime.
